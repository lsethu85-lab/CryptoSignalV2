Logic / Documentation & Rules (Institutional Swing Framework)

This tab documents the exact mathematical workflow used by the dashboard to produce Buy / Hold / Trim / Sell signals, enforce liquidity guardrails, compute Hard Stop-Loss levels, and rank Momentum Quality Picks.

A) Timeframe Separation (Non‑negotiable)

1D = Macro regime & swing structure (trend + key support/resistance).
4H = Momentum confirmation (EMA21 / RSI / MACD + volume validation).
1H = Execution buffer only (not used for trend decisions).

B) Liquidity & Anti‑Noise Guardrail

Assets are automatically rejected from Momentum Quality Picks unless they pass:
market_cap ≥ $1B AND 24h_volume ≥ $50M AND (24h_volume/market_cap) ≥ 1.5%
If failed → signal becomes SKIP (Illiquid) and the setup is excluded from picks.

C) Signal Matrix — Buy / Hold / Trim / Sell

Each coin receives factor flags. The action is the highest‑priority rule that matches.
1) Macro Trend (1D)
SMA50 = SMA(close_1D,50) · SMA200 = SMA(close_1D,200)
MacroUp = (close_1D > SMA50) AND (SMA50 > SMA200)
MacroDown = (close_1D < SMA50) AND (SMA50 < SMA200)
2) Local Momentum (4H)
EMA21_4H = EMA(close_4H,21)
RSI_4H = RSI(close_4H,14) → Bull zone: 55–70 · Bear: <45
MACD = EMA12−EMA26; Signal=EMA(MACD,9)
MACDUpCross = (MACD_now > Signal_now) AND (MACD_prev ≤ Signal_prev)
3) Volume Validation (4H)
VMA20_4H = SMA(volume_4H,20)
VolSpike = volume_4H_now ≥ 1.35 × VMA20_4H
Breakout = close_4H_now > Highest(close_4H,20)
VolumeConfirm = Breakout AND VolSpike
4) DeFi Fundamental Layer (TVL)
TVL/MC = TVL_current / market_cap_current
TVL_7d_change = (TVL_now − TVL_7d_ago) / TVL_7d_ago
TVL_Bull = (TVL_7d_change ≥ +5%) OR (TVL/MC rising vs 7d avg)
TVL_Bear = (TVL_7d_change ≤ −5%)
Action Rules (priority order)
SELL if MacroDown AND (close_4H < EMA21_4H OR RSI_4H < 45 OR TVL_Bear)
BUY if MacroUp AND close_4H > EMA21_4H AND RSI_4H ∈ [55,70] AND MACDUpCross AND VolumeConfirm AND TVL_Bull AND RR ≥ 2.0
TRIM if MacroUp AND (RSI_4H > 70 OR funding_rate_high OR distance_to_resistance < 0.5×ATR_1D OR TVL_Bear)
HOLD otherwise (including neutral macro regimes)

D) Risk/Reward Calculus (2:1+ gate)

Key levels come from clustered pivot zones on 1D. The dashboard computes:
Stop = (NearestSupport − 0.25×ATR_1D) (Hard stop below structure)
TakeProfit (TP2) = Entry + 2×(Entry − Stop) (2R target)
RR = (TP2 − Entry) / (Entry − Stop)
No asset can enter Momentum Quality Picks unless RR ≥ 2.0 (optionally 3.0 if you raise the toggle in settings).

E) Position Sizing Protocol (1–2% risk)

Risk per trade is capped.
Risk$ = AccountEquity × RiskPct (RiskPct = 0.01 to 0.02)
StopDistance$ = |Entry − Stop|
PositionSize (units) = Risk$ / StopDistance$
PositionNotional$ = PositionSize × Entry

F) Momentum Quality & Holding Horizon

Expected holding horizon is a function of daily volatility (ATR) and distance to resistance:
ExpectedBars ≈ (Resistance − Entry) / ATR_1D
Gauge = 1–3 Days (≤3) · 3–5 Days (≤5) · 1–2 Weeks (≤14) · 2–4 Weeks (≤30) · 1–3 Months (>30)
Funding-rate warning: if funding rate is persistently positive above a threshold, the dashboard flags "crowded longs" for long-held trades.
